# Local Information Tracker (lit)

**lit** は、「ローカルで完結するシンプルかつ強力なバージョン管理システム (VCS)」です。
ユーザーによる手動コミットを排除し、ファイルの変更を検知して全自動でスナップショットを作成する仕組みを提供します。また、Gitでは扱いにくい大容量ファイルやバイナリファイルも効率的に管理できるように設計されています。

## コンセプト

* **シンプル**: コマンド数を最小限に抑え、直感的な操作性を実現。
* **自動化**: 変更検知による自動スナップショット作成（常駐デーモンによる監視）。
* **大容量対応**: Content-Defined Chunking (CDC) により、巨大なファイルやバイナリも効率的に保存。

##  機能一覧

### CLI機能 (litコマンド)

| コマンド | 説明 |
| :--- | :--- |
| `lit add <filepath>` | ファイルを追跡対象に追加します（カレントディレクトリ内のみ）。|
| `lit remove <filepath>` | ファイルを追跡対象から除外します。|
| `lit list` | 追跡中のファイル一覧を表示します。|
| `lit log <filepath>` | 指定したファイルのコミット（スナップショット）履歴を表示します。|
| `lit restore <commitID>` | ファイルを特定のコミットIDの状態に復元します。|
| `lit gc` | ガベージコレクション（デフラグ）を実行し、ストレージを最適化します。(実装予定)|

### バックグラウンド機能 (lit-watcher)

* **ファイル監視**: `fsnotify` を使用し、追跡対象ファイルの変更を常時監視します。
* **自動スナップショット**: 変更検知後、デバウンス処理を経て自動的にスナップショットを作成・保存します。
